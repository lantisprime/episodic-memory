## §3 Objective

Surface the existing K/N consolidation-cadence advisory at session start (RFC-012 R3a):
`em-trigger-index.mjs` computes the two gauges per store at build time (phrase-sharing
>= K=3 over phrase-kind entries; active `category: lesson` rows >= N=200), stamps a typed
`session_start.cadence` field into `trigger-index.json`, and the session-start renderer
emits at most one budgeted advisory line. `em-consolidate`'s REQ-23 advisory switches to
the same shared computation, correcting its N gauge from count-all-eligible to
lesson-only. Provable by: gauge-threshold fixtures both polarities per gauge + an
envelope truncation-order fixture (RFC-012:155).

## §8 Design

### 8.1 Key types

```js
/**
 * @typedef {Object} CadenceField — session_start.cadence (schema v4, closed)
 * @property {true} enabled — build-compiled enablement flag (R3a); constant in P1
 * @property {number} phrase_sharing — max phrase-kind entries sharing one value (>=0)
 * @property {number} active_lessons — active, non-expired category:lesson rows (>=0)
 * @property {3} k_shared — threshold echoed for observability (CADENCE_K_SHARED)
 * @property {200} n_lessons — threshold echoed (CADENCE_N_LESSONS)
 * @property {string} [line] — the one advisory line; PRESENT iff a gauge fired
 */
```

### 8.2 Key invariants

- Advisory-only (B-1, RFC-008 R1): the field and line are never read by any gate.
- Per-store purity (R3a): each store's stamp derives from that store's rows+entries only;
  no env inputs (mirrors buildSessionStart, em-trigger-index.mjs:784-785).
- One line max, session-start only, truncate-first on envelope overflow (R3a).
- Exact advisory strings byte-identical to today's (em-consolidate.mjs:588,590) — no
  string churn for downstream matchers.
- **Cross-platform:** pure JS date/string ops; no shell, no GNU flags; test fixtures via
  `os.tmpdir()` + `path`.
- **Atomicity:** unchanged — the existing unique-temp+rename write persists the index.

### 8.3 Resolution / flow and the settled decisions

```text
buildTriggerIndex(scope) -> rows, entries -> computeCadence -> session_start.cadence
loadMergedTriggerIndex -> merged session_start + LOCAL cadence threaded
renderSessionStart -> [...tiers, ...playbooks, ...tier2, ...preflight, cadence?, patternHealth?]
em-consolidate --clerk -> computeCadence(mergedEntries, storeRows) -> report.advisory
```

- **D1 always-stamp:** `cadence` is stamped on every successful build (both scopes) with
  gauge values; `line` alone is conditional. Gauge observability is the typed surface;
  presence-of-line is the fired state. Both polarities are therefore schema-testable.
  Deliberately NOT flag-gated: the planner's runtime probe showed pattern_health is
  absent from the hook's own invocation today because the hook never passes
  `--with-pattern-health` (activation-hook-run.mjs `--scope`-only args) — flag-gating
  would make R3a a dead letter on the exact surface it targets.
- **D2 shape:** as §8.1; thresholds echoed so a consumer never needs the lib constants.
- **D3 render placement:** after preflight, before pattern-health. R3a fixes the RFC-012
  advisory order (cadence, stagnation, promotion, needs-enforcement); pattern-health is
  today's carrier of the `needs-enforcement` verdict (activation-match.mjs:652), so
  cadence must precede it. Sequential budgeting means later blocks drop first; cadence
  sits after all lesson pointers, satisfying "advisories truncate first".
- **D4 merged view threads LOCAL only — at BOTH merge-site classes:** exact
  pattern_health precedent, which exists in THREE places: `loadMergedTriggerIndex`
  (em-trigger-index.mjs:971-973, CLI/clerk path) and each deployed activation hook's
  own `mergeSessionStart` (claude-code-activation :443-444, codex-activation :418-419)
  — the hooks never call `loadMergedTriggerIndex` (GLM r1 F1, probe-verified: only
  comment refs at activation-hook-run.mjs:277,287,389). All three get the cadence
  thread. The global store's stamp lives in its own artifact with no merged-view
  reader in P1 — deliberate forward surface: R3a mandates per-store gauges and
  R3c/P3 (promotion advisory over registered stores) is the named future consumer.
- **D5 clerk parity — with a documented scope asymmetry (planner P-3):** em-consolidate
  keeps its merged-entries phrase gauge input (existing behavior, :469) and switches N
  to the corrected lesson filter via the shared function. The clerk's `phrase_sharing`
  is therefore a CROSS-STORE (merged) quantity while the session-start stamp's is
  PER-STORE — same field names, intentionally different scope, because the clerk
  advises on the corpus it consolidates (merged) and R3a pins the stamp to the store.
  The two can legitimately diverge (`testMergedDivergence` proves it); consumers MUST
  NOT equate the clerk report's advisory with a given store's stamped line.
- **D6 schema v4:** additive closed-schema bump, exact v2->v3 playbooks precedent
  (em-trigger-index.mjs:56-63); contract mirror bumped in lockstep.

## §12 Contracts

### `computeCadence(entries, rows, now) → CadenceField`

**Input contract:** `entries` any array (non-arrays treated as `[]`); `rows` any array
(non-arrays treated as `[]`); malformed members skipped, never thrown on. `now` a Date
(defaults to `new Date()`).
**Output contract:** always returns a `CadenceField` (§8.1); `line` key present iff a
gauge fired; never null, never throws.

**State table (exhaustive):**

| State | Condition | Output | Side effects |
|---|---|---|---|
| A. K fires | `phrase_sharing >= 3` (regardless of N) | `line` = phrase-sharing string with observed count | none |
| B. N fires | `phrase_sharing < 3` AND `active_lessons >= 200` | `line` = active-lessons string with observed count | none |
| C. quiet | `phrase_sharing < 3` AND `active_lessons < 200` | no `line` key; gauge values present | none |
| D. empty inputs | `entries`/`rows` empty or not arrays | state C with both gauges 0 | none |
| E. malformed members | null rows, rows without category/status, entries without value | skipped from counts; remainder counted | none |
| F. expired lesson | `review_by < today` (string compare, same as em-trigger-index.mjs:684) | excluded from `active_lessons` | none |
| G. non-lesson actives | e.g. 500 active `decision` rows, 0 lessons | `active_lessons = 0`; N gauge quiet (THE correction) | none |

**Error codes:** none — the function cannot fail (fail-open to quiet); the STAMP site
wraps in try/catch anyway (defense in depth, REQ-9) and emits
`em-trigger-index: cadence gauge failed (<msg>); omitted` on stderr.
