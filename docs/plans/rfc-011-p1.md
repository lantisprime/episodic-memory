## §8 Design

Normative design lives in RFC-011 R1-R7 (revision `9be7c93`). Plan-level invariants:

- **Cross-platform:** all new code is zero-dep Node ESM; paths via `path`/`os`; no shell-outs added; the `--read` output bound binds the SERIALIZED body (`JSON.stringify(body).length <= 49152`, REQ-14), not terminal- or platform-dependent.
- **Atomicity:** trigger-index writes keep the existing temp+rename (em-trigger-index.mjs:295-303); no new shared-state writers. `--read` tracking reuses em-search's existing index write-back path.
- **Fail directions:** advisory surfaces fail open to NOTHING; retention fails CLOSED (abort). Never mix.
- **One new export per lib maximum; thin additions over refactors** (blast-radius pattern §A.9): `matchActivation`/`renderSessionStart` gain branches, not restructures.

## §12 Contracts (state tables)

### `parsePlaybooksConfig(storeDir) → {ok, config?, reason?}` (new shared helper; single parser for build + protection)

| State | Condition | Output | Side effects |
|---|---|---|---|
| A absent | no playbooks.json | `{ok:true, config:null}` | none |
| B valid | parses + schema-valid + ≤32 entries + ≤64KiB | `{ok:true, config}` | none |
| C malformed | present but unparseable/invalid/over-bound | `{ok:false, reason:<string>}` | none (CALLERS decide fail direction: build → skip+note; prune/consolidate → abort exit 1) |

### `--read <id>` (em-search)

| State | Condition | Output | Side effects |
|---|---|---|---|
| A found | id exists in local or global store | `{status:"ok", episode:{...full}}` exit 0 | access_count+1, last_accessed=now on that row (unless `--no-track`) |
| B missing | id in neither store | `{status:"error", message:...}` exit 1 | none |
| C with --no-track | as A | as A | none |

### Build playbook resolution (per entry)

| State | Condition | Output |
|---|---|---|
| A resolved | terminal active lesson, no collision | derived row(s) + declared listing |
| B-H excluded | unresolvable / cycle / inactive / non-lesson / expired / chain-collision / empty-triggers | excluded counter++ (named key), no row, never fatal |

Fail direction (gates): none of this gates — advisory surfaces fail open to nothing; ONLY the retention consumers fail closed (abort archival).
