## §8 Design

Two event hooks + one SessionStart hook + one pure core, reading only the persisted per-store
`trigger-index.json` files. Obeys the RFC two-plane contract: no model in the event plane, no decision
channel, only the derived index crosses the boundary — and **everything the event plane needs is baked into
that index at build time** (per-store bands, activity phrase sets).

```mermaid
sequenceDiagram
    participant U as Prompt / Tool / SessionStart event
    participant H as activation hook (event plane)
    participant L as local trigger-index.json
    participant G as global trigger-index.json
    participant CORE as matchActivation (pure)
    U->>H: stdin JSON (cwd, prompt/tool/…)
    H->>L: stat freshness; FRESH read / STALE rebuild (carve-out) + read
    H->>G: stat freshness; FRESH read / STALE rebuild + read
    Note over H: merge in-core, dedup by episode_id, LOCAL precedence (RFC L73)
    H->>CORE: matchActivation(merged, event, identity, suppress, bounds)
    CORE-->>H: bounded rendered lines (NO decision field)
    H-->>U: additionalContext (exit 0, always)
```

### 8.1 Key types

```js
/**
 * @typedef {Object} ActivationEvent
 * @property {'prompt'|'tool'|'session_start'} kind
 * @property {string} [prompt]  — kind==='prompt'
 * @property {string} [tool]    — kind==='tool': tool name
 * @property {string} [target]  — kind==='tool': command | file_path | '' (unknown tool)
 *
 * @typedef {Object} MatchResult
 * @property {string[]} lines
 * @property {string|null} overflowNote  — '+N … incl. critical <id>' (states H AND I)
 * // INVARIANT: no decision/block/permissionDecision field; core throws nothing (fails open to empty).
 */
```

### 8.2 Key invariants

- **SYMMETRY (the level-2 governor).** Every invariant below applies to **every hook and every branch,
  symmetrically** — the P1b asymmetric-validation/proto-key class (`…3c55`). Tests derive from this clause,
  not per-hook enumeration; a constraint on one hook/branch with a lenient sibling is a bug.
- **Advisory-only:** the core and all three hooks emit no decision field and always exit 0 — including the
  stale-build-throw, corrupt-index, and suppress-error branches.
- **Event-plane reads ONLY persisted `trigger-index.json`:** never `index.jsonl`, `tags.json`, bodies,
  `activation-classes.json`, or environment (`EM_ACTIVATION_CLASSES_PATH` included). Fresh path = stat-only
  freshness. STALE path = the per-store R2 build under the carve-out (reads `index.jsonl` as BUILD input;
  output flows only through the rebuilt artifact). The core does NOT call `loadMergedTriggerIndex` (it reads
  `index.jsonl` live — aggregation-side only).
- **Everything baked at build:** activity phrase sets (REQ-9) and per-store `effective_priority` bands live
  in the index; the event plane reads them, never re-derives the band, never resolves classes at event time.
- **Cross-tier dedup:** R4 tier-2 excludes any `episode_id` already emitted in tier-1; tier-1 wins.
- **Tier-2 is plain-only:** `session_start.entries` carry no `effective_priority` (probe-verified); no code
  reads `entry.effective_priority` (undefined) — tier 2 renders `lesson <id>: <summary>` unconditionally.
- **Preflight keys are em-recall TASK TYPES** (`implementation`/`push`/`rule`), not activity classes
  (`implement`); R4 derives `count=sum(values)`, `ids=keys` generically and never cross-maps to activity classes.
- **Suppress fails OPEN, whole-file:** any failure (missing/syntax/shape/schema_version) → empty Set + one
  stderr note. Mute is by episode id, all bands.
- **Scope predicate:** `applies_to.includes(project) || applies_to.includes('*')`; empty `[]` → never fires.
- **No `obj[key]`-as-membership** anywhere (`Map`/`Set`/`Object.hasOwn`).
- **Cross-platform:** bash+`jq` hooks; zero-dep Node core; realpath both sides on path predicates; no GNU-only flags.
- **Atomicity:** the STALE-path rebuild is atomic temp+rename (P1b guarantee).

### 8.3 Resolution / flow

```text
stdin JSON → parse (empty/non-object → exit 0, no output)
  → for each store: stat index.jsonl vs trigger-index.json.source fingerprint
      FRESH → read the trigger-index.json file
      STALE → run `em-trigger-index --scope <s> --project <root>` (carve-out) → read the rebuilt file
  → merge the two files' entries + session_start in-core (dedup by id, LOCAL precedence)
  → read lesson-suppress.json (whole-file try/catch → Set | empty)
  → matchActivation(merged, event, identity, suppress, {max_matches:3, max_tokens:500})
  → additionalContext = lines.join('\n') (+ overflowNote) → emit → exit 0
```

## §12 Contracts

### `matchActivation(index, event, identity, suppress, bounds) → MatchResult`

Fails **open** (empty result) on any internal error; throws nothing; never a decision field.

| State | Condition | Output | overflowNote |
|---|---|---|---|
| A. no index | entries absent/empty | `{lines:[]}` | null |
| B. no match | none matches event+identity | `{lines:[]}` | null |
| C. plain | matches all ≤7, ≤ max_matches | plain lines | null |
| D. imperative | a match is 8–9 | imperative line | null |
| E. suppressed | id ∈ suppress (any band) | absent | null |
| F. scope miss | applies_to excludes identity (incl. empty `[]`) | absent | null |
| G. overflow, no crit dropped | > max_matches, none dropped is 8–9 | top-K | null |
| H. overflow, crit dropped | a dropped match is 8–9 | top-K | `+N … incl. critical <id>` |
| **I. oversize crit dropped** | a band-8/9 entry's line > max_tokens | that entry dropped | **`+N … incl. critical <id>`** (F5) |
| I′. oversize non-crit | a ≤7 entry's line > max_tokens | dropped | null |
| J. malformed entry | missing value/id | skipped | null (stderr note) |
| K. proto-key class | `activity:__proto__` etc. | no match | null |

**Fail direction:** the advisory core fails **open** (empty result) — the deliberate inverse of a gate.
This is correct because the layer is advisory (RFC-008 R1); a fail-closed advisory that blocked a tool call
would be the bug.
